# Анализ данных игровой индустрии (2000–2013)

Проект выполнен для команды онлайн-игры с целью подготовки статьи о развитии игровой индустрии в начале XXI века. В ходе исследования проведена предобработка исторических данных о продажах игр, их жанрах, платформах и оценках. Основной акцент сделан на играх жанра RPG, однако данные охватывают все жанры. Результаты помогут коллегам привлечь новую аудиторию через публикацию на известном IT-ресурсе.

## Цели проекта

- Познакомиться с предоставленными данными, проверить их корректность и провести предобработку.
- Отобрать игры, выпущенные с **2000 по 2013 год** включительно.
- Категоризовать игры по оценкам пользователей и критиков (высокая, средняя, низкая, нет оценки).
- Выделить **топ-7 платформ** по количеству выпущенных игр за указанный период.

## Описание данных

Исходный датасет: `data/new_games.csv`

| Поле | Описание |
|------|----------|
| `Name` | Название игры |
| `Platform` | Платформа |
| `Year of Release` | Год выпуска |
| `Genre` | Жанр |
| `NA sales` | Продажи в Северной Америке (млн копий) |
| `EU sales` | Продажи в Европе (млн копий) |
| `JP sales` | Продажи в Японии (млн копий) |
| `Other sales` | Продажи в других странах (млн копий) |
| `Critic Score` | Оценка критиков (0–100) |
| `User Score` | Оценка пользователей (0–10) |
| `Rating` | Рейтинг ESRB (возрастная категория) |

Данные собраны из открытых источников и охватывают период активного роста индустрии: появление PlayStation 2, Xbox, Nintendo Wii, развитие многопользовательских игр и сообществ.

## Этапы работы

### 1. Загрузка и знакомство с данными
- Импорт библиотек (pandas, numpy).
- Чтение CSV, первичный осмотр структуры, типов данных, пропусков.
- Вывод первых строк и метода `info()`.

### 2. Предобработка данных
- **Названия столбцов:** приведены к стилю `snake_case` (нижний регистр, пробелы заменены на `_`).
- **Типы данных:**  
  - `eu_sales`, `jp_sales`, `user_score` — замена строковых значений ('unknown', 'tbd') на `NaN` и приведение к `float`.  
  - `year_of_release` — после обработки пропусков приведён к целочисленному типу.
- **Обработка пропусков:**  
  - Пропуски в `eu_sales` и `jp_sales` заменены на средние значения по группе (платформа + год).  
  - Строки с пропусками в `name`, `genre`, `year_of_release` удалены (менее 2% данных).  
  - Пропуски в `critic_score`, `user_score`, `rating` (более 50%) оставлены — для них введена категория «нет оценки».
- **Дубликаты:**  
  - Приведение значений в `genre` к нижнему регистру для устранения неявных дубликатов.  
  - Удалено 235 явных дубликатов.  
- **Итог:** удалено 512 строк (≈3% от исходного объёма).

### 3. Фильтрация данных
- Отобраны игры, выпущенные в период **2000–2013** (включительно).  
- Сохранён срез в отдельный датафрейм `df_actual` (12781 строк).

### 4. Категоризация оценок
- **Пользовательские оценки (`user_score`)**  
  - Высокая: от 8 до 10 включительно  
  - Средняя: от 3 до 8 (не включая 8)  
  - Низкая: от 0 до 3 (не включая 3)  
  - Для игр без оценки добавлена категория «нет оценки»  
- **Оценки критиков (`critic_score`)**  
  - Высокая: от 80 до 100 включительно  
  - Средняя: от 30 до 80 (не включая 80)  
  - Низкая: от 0 до 30 (не включая 30)  
  - Для игр без оценки добавлена категория «нет оценки»  
- Добавлены столбцы `user_category` и `critic_category`.

### 5. Топ-7 платформ по количеству игр
- Подсчёт количества игр на каждой платформе и выделение лидеров.
